## UI Components

ZT-UI Components provides a rich set of components organized into the following categories:

### Form Components
- **`<zt-input>`** - Advanced text input with validation, character counting, and multiple visual styles
- **`<zt-textarea>`** - Multi-line text input with length validation and character counting
- **`<zt-select>`** - Dropdown select with data binding, search capabilities, and customizable styles

### Interactive Components
- **`<zt-button>`** - Versatile button with variants, sizes, states, and interaction handling
- **`<zt-modal>`** - Flexible modal dialogs with theming, accessibility, and customizable content
- **`<zt-paginator>`** - Pagination component for navigating data tables and lists
- **`<zt-toast>`** - Toast notification system with multiple types, positions, and customizable options

### Layout Components
- **`<zt-card>`** - Flexible container with headers, bodies, footers, variants, and hover effects

### Data Display Components
- **`<zt-data-grid>`** - Advanced data table with sorting, filtering, pagination, and row selection

## Quick Start

Let's get you up and running with your first ZT-UI Components in just a few minutes.

### 1. Import Components

For standalone Angular applications (recommended):

```typescript
import { Component } from '@angular/core';
import { ZtButton, ZtInput, ZtCard, ZTThemeService } from 'zt-ui-components';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [ZtButton, ZtInput, ZtCard],
  template: `
    <zt-card cardStyle="elevated" size="md">
      <div class="card-header">
        <h3>Welcome to ZT-UI</h3>
      </div>
      <div class="card-body">
        <zt-input
          placeholder="Enter your name"
          size="zt-md"
          [textlength]="50">
        </zt-input>
        <zt-button variant="primary" size="zt-md">
          Get Started
        </zt-button>
      </div>
    </zt-card>
  `,
})
export class AppComponent {
  constructor(private themeService: ZTThemeService) {
    // Set a theme (optional)
    this.themeService.setThemeByName('light');
  }
}
```

For module-based applications:

```typescript
// app.module.ts
import { NgModule } from '@angular/core';
import { BrowserModule } from '@angular/platform-browser';
import { ZtButtonModule, ZtInputModule, ZtCardModule } from 'zt-ui-components';

@NgModule({
  declarations: [AppComponent],
  imports: [
    BrowserModule,
    ZtButtonModule,
    ZtInputModule,
    ZtCardModule,
  ],
  bootstrap: [AppComponent]
})
export class AppModule { }
```

### 2. Add Global Styles

Include the component styles in your global stylesheet:

```scss
// styles.scss
@import '~zt-ui-components/styles';
```

### 3. Configure Theme (Optional)

Set up a global theme in your `app.config.ts`:

```typescript
import { ApplicationConfig } from '@angular/core';
import { THEME_CONFIG } from 'zt-ui-components';

export const appConfig: ApplicationConfig = {
  providers: [
    {
      provide: THEME_CONFIG,
      useValue: {
        name: 'custom-brand',
        colors: {
          primary: '#007bff',
          success: '#28a745',
          danger: '#dc3545',
        },
        borderRadius: 8,
      }
    }
  ]
};
```

### Reactive Forms Integration

ZT-UI Components work seamlessly with Angular Reactive Forms:

```typescript
import { FormBuilder, FormGroup, Validators } from '@angular/forms';

export class ContactFormComponent {
  contactForm: FormGroup;

  constructor(private fb: FormBuilder) {
    this.contactForm = this.fb.group({
      name: ['', [Validators.required, Validators.minLength(2)]],
      email: ['', [Validators.required, Validators.email]],
      message: ['', [Validators.required, Validators.maxLength(500)]]
    });
  }

  onSubmit() {
    if (this.contactForm.valid) {
      console.log('Form data:', this.contactForm.value);
    }
  }
}
```

### Toast Notifications

Toast notifications provide non-intrusive feedback to users about actions, errors, or information:

```html
<!-- Basic toast component -->
<zt-toast position="top-right"></zt-toast>
```

```typescript
import { ZtToastService } from 'zt-ui-components';

export class NotificationComponent {
  constructor(private toastService: ZtToastService) {}

  performAction() {
    // Simulate an action
    this.toastService.success('Action completed successfully!', 'Operation Complete', {
      duration: 3000,
      dismissible: true,
      showProgress: true
    });
  }

  handleError() {
    this.toastService.error('Something went wrong!', 'Error Occurred', {
      duration: 5000,
      dismissible: true
    });
  }

  showInfo() {
    this.toastService.info('Here is some useful information.', 'Information');
  }

  showWarning() {
    this.toastService.warning('Please review your input.', 'Caution');
  }
}
```

#### Toast Types

- **Success**: Green styling with checkmark icon
- **Error**: Red styling with error icon
- **Info**: Blue styling with info icon
- **Warning**: Yellow styling with warning icon

## Theming System

### Global Theme Setup

Set a global theme that applies to all components:

```typescript
import { ZTThemeService, LIGHT_THEME } from 'zt-ui-components';

@Component({...})
export class AppComponent implements OnInit {
  constructor(private themeService: ZTThemeService) {}

  ngOnInit() {
    // Set predefined theme
    this.themeService.setThemeByName('dark');

    // Or set custom theme
    this.themeService.setTheme({
      name: 'brand-theme',
      colors: {
        primary: '#FF6B35',
        success: '#4CAF50',
        danger: '#F44336',
        textBlack: '#2D3748',
        default: '#FFFFFF',
      },
      borderRadius: 12,
      borderSize: 2,
      fontFamily: '"Inter", sans-serif'
    });
  }
}
```

### Local Theme Overrides

Override theme properties for individual components:

```html
<!-- Component with local theme override -->
<zt-card [ztTheme]="cardTheme" cardStyle="elevated">
  <div class="card-body">
    <h3>Custom Themed Card</h3>
    <zt-input
      [ztTheme]="inputTheme"
      placeholder="Custom styled input">
    </zt-input>
  </div>
</zt-card>
```

```typescript
export class MyComponent {
  cardTheme: Partial<ThemeConfig> = {
    colors: {
      primary: '#FF6B35',
      borderRadius: 16,
    },
    borderSize: 3
  };

  inputTheme: Partial<ThemeConfig> = {
    colors: {
      primary: '#4ECDC4',
      textBlack: '#2C3E50'
    },
    borderRadius: 8
  };
}
```

### Dynamic Theme Switching

Implement runtime theme switching:

```typescript
export class ThemeSwitcherComponent {
  availableThemes = ['light', 'dark', 'bootstrap', 'material'];
  currentTheme$ = this.themeService.currentTheme$;

  constructor(private themeService: ZTThemeService) {}

  setTheme(themeName: string) {
    this.themeService.setThemeByName(themeName as any);
  }

  toggleDarkMode() {
    this.themeService.toggleDarkMode();
  }
}
```

| Property | Type | Default | Description |
|----------|------|---------|-------------|

| `showDemoButtons` | `boolean` | `false` | Show demo buttons for testing |
| `ztTheme` | `Partial<ThemeConfig>` | - | Local theme override |

**Toast Service Methods:**
- `success(message: string, title?: string, options?: Partial<ZtToast>)` - Show success toast
- `error(message: string, title?: string, options?: Partial<ZtToast>)` - Show error toast
- `info(message: string, title?: string, options?: Partial<ZtToast>)` - Show info toast
- `warning(message: string, title?: string, options?: Partial<ZtToast>)` - Show warning toast
- `show(toast: Omit<ZtToast, 'id'>)` - Show custom toast
- `remove(id: string)` - Remove specific toast
- `clear()` - Remove all toasts

**Toast Options:**
- `duration?: number` - Auto-hide duration in milliseconds (default: 5000)
- `dismissible?: boolean` - Allow manual dismissal (default: true)
- `showProgress?: boolean` - Show progress bar (default: false)
- `position?: ToastPosition` - Toast position
- `title?: string` - Toast title
- `type: 'success' \| 'error' \| 'info' \| 'warning'` - Toast type

## Troubleshooting

### Common Issues and Solutions

#### 1. Components Not Rendering

**Problem:** Components appear as plain HTML elements.

**Solution:** Ensure components are properly imported:

```typescript
// For standalone components
imports: [ZtButton, ZtInput, ZtCard]

// For modules
imports: [ZtButtonModule, ZtInputModule, ZtCardModule]
```

#### 2. Theme Not Applying

**Problem:** Theme changes don't take effect.

**Solutions:**
- Check that `ZTThemeService` is injected correctly
- Ensure theme is set before component initialization
- Verify theme configuration is valid

```typescript
// Correct theme application
constructor(private themeService: ZTThemeService) {}

ngOnInit() {
  this.themeService.setThemeByName('dark');
}
```

#### 3. Form Validation Issues

**Problem:** Form controls not validating properly.

**Solution:** Ensure proper `ControlValueAccessor` implementation:

```typescript
// In component with reactive forms
this.form = this.fb.group({
  email: ['', [Validators.required, Validators.email]]
});
```

```html
<zt-input
  formControlName="email"
  placeholder="Email"
  inputType="email">
</zt-input>
```

#### 4. Performance Issues

**Problem:** Components causing performance problems.

**Solutions:**
- Use `OnPush` change detection
- Avoid complex expressions in templates
- Implement proper trackBy functions for `*ngFor`

#### 5. Accessibility Warnings

**Problem:** Screen reader or keyboard navigation issues.

**Solution:** Ensure proper ARIA attributes:

```html
<zt-input
  placeholder="Search"
  aria-label="Search products"
  [textlength]="100">
```

### Adding New Components

Follow this step-by-step process:

1. **Generate Component Structure**
   ```bash
   ng generate component zt-new-component --project=zt-ui-components --style=scss --standalone
   ```

2. **Implement Component Logic**
   ```typescript
   @Component({
     selector: 'zt-new-component',
     standalone: true,
     imports: [CommonModule],
     templateUrl: './zt-new-component.component.html',
     styleUrls: ['./zt-new-component.component.scss'],
     changeDetection: ChangeDetectionStrategy.OnPush,
   })
   export class ZtNewComponent implements OnInit {
     @Input() title = '';
     @Input() size: 'sm' | 'md' | 'lg' = 'md';
     @Input() ztTheme?: Partial<ThemeConfig>;
   }
   ```

3. **Add Theme Support**
   ```scss
   @use "../../assets/themes" as themes;

   :host {
     --zt-component-bg: var(--zt-default, #ffffff);
     --zt-component-color: var(--zt-text-black, #212529);
   }
   ```

4. **Write Unit Tests**
   ```typescript
   describe('ZtNewComponent', () => {
     it('should create', () => {
       expect(component).toBeTruthy();
     });
   });
   ```

5. **Create Storybook Stories**
   ```typescript
   const meta: Meta<ZtNewComponent> = {
     title: 'Components/New Component',
     component: ZtNewComponent,
     tags: ['autodocs'],
   };
   ```

6. **Update Documentation**
   - Add component to README.md
   - Update API reference
   - Add usage examples
